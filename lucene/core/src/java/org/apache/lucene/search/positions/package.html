The positions package provides mechanisms to enable querying using minimum interval semantics, as defined in "Efficient Optimally Lazy Algorithms for Minimal-Interval Semantics

The following Query implementations are provided:

  1. {@link org.apache.lucene.search.positions.IntervalFilterQuery - IntervalFilterQuery} -- Filter a Query based on the positions or ranges of its component parts
  2. {@link org.apache.lucene.search.positions.OrderedNearQuery - OrderedNearQuery} -- Filter queries based on the ordered difference between their match positions in a document
  3. {@link org.apache.lucene.search.positions.UnorderedNearQuery - UnorderedNearQuery} -- Filter queries based on the unordered difference between their match positions in a document
  4. {@link org.apache.lucene.search.positions.NonOverlappingQuery - NonOverlappingQuery} -- Match queries whose match positions do not overlap
All queries require positions to be stored in the index.
